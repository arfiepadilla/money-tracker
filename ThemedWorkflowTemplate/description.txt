Themed Workflow Template - A comprehensive copy-paste template demonstrating all common UI components with proper color theming using pure Tailwind CSS.

This template serves as a starting point for creating new workflows with consistent styling. It uses a single color family (currently cyan) throughout, making it easy to find-and-replace the color name to match your workflow's purpose. The template includes four tabs showcasing buttons, form elements, components, and state indicators.

Perfect for developers who want to quickly bootstrap a new workflow with professional, cohesive styling without having to manually create each UI component from scratch.

Key features demonstrated:
- Complete button system (primary, secondary, outline, success, danger, warning, disabled)
- Multiple button sizes (xs, sm, md, lg) with proper styling
- Icon buttons with hover states
- All form elements (text input, textarea, select, checkbox, radio)
- Card/panel components with proper borders and backgrounds
- Badges and tags with different semantic meanings
- List items with hover states and descriptions
- Status indicators with semantic colors and animations
- Alert boxes for success/info/warning/error messages
- Loading states (spinners, progress bars)
- Tab navigation with active state handling

Template usage instructions:
1. Copy the entire file to your new workflow directory
2. Choose a color family (purple, cyan, emerald, amber, slate, pink, red, orange, lime, indigo)
3. Find and replace 'cyan' with your chosen color throughout the file
4. Customize the content and functionality for your specific use case
5. Test to ensure good contrast and readability

Color scheme structure:
- {color}-950: Darkest background (main container)
- {color}-900: Secondary background (cards, panels)
- {color}-800: Tertiary background (secondary buttons, hover states)
- {color}-700: Borders and dividers
- {color}-600: Primary buttons and accents
- {color}-500: Button hover states
- {color}-400: Muted text and placeholders
- {color}-200: Secondary text
- {color}-100: Primary text

Best practices included:
- Semantic colors always used consistently (green=success, red=error, yellow=warning, blue=info)
- Focus states with ring-2 for accessibility
- Proper hover transitions on interactive elements
- Disabled states with opacity-50 and cursor-not-allowed
- Responsive design with grid layouts
- Consistent spacing with Tailwind utility classes

WHEN TO USE:
- Starting a new workflow from scratch
- Need a reference for proper Tailwind theming patterns
- Want consistent styling across your workflow ecosystem
- Building professional tools with focused, cohesive aesthetics

This template emphasizes simplicity, consistency, and ease of customization. By using a single color family, it maintains a clean, professional look that's perfect for productivity tools, developer tools, data analysis interfaces, and other focused applications.
