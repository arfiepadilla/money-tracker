Multi-Color Workflow Template - A comprehensive template demonstrating how to use multiple color families (blue, purple, emerald, pink) effectively in a single workflow.

This template features a dashboard layout with tabbed navigation, where each section uses its own dedicated color theme while maintaining a cohesive dark slate base. It includes metric cards with color-coded categories, status indicators with semantic colors, and section-specific theming.

Perfect for building dashboards, creative tools, or applications with distinct feature sections that benefit from visual color coding. Shows best practices for balancing multiple accent colors while keeping the design professional and not overwhelming.

Key features demonstrated:
- Neutral slate base (bg-slate-950/900) that lets accent colors pop
- Purpose-driven color assignment (blue=data, purple=AI, emerald=analytics, pink=creative)
- Consistent semantic colors for status (green=success, yellow=warning)
- Semi-transparent colored backgrounds (/30, /50 opacity) for subtle sections
- Color-coded navigation tabs and metric cards
- Section-specific theming that maintains consistency

Best practices included:
- Limit to 3-5 accent colors maximum
- Use neutral base to avoid visual chaos
- Assign each color a specific meaning/purpose
- Keep semantic colors consistent throughout
- Use opacity for backgrounds to avoid overwhelming the user

WHEN TO USE:
- Dashboards with multiple distinct metrics or sections
- Creative tools where different modes need visual distinction
- Apps with multiple feature categories
- Entertainment or gaming interfaces

WHEN NOT TO USE:
- Professional focused tools (documents, terminals) where calm aesthetics matter
- Applications where cohesive single-color branding is more important
- When minimalism and visual calm are priorities
